# Use check for setup inspection

PullOps setup commands use `--check`, not `--dry-run`, for read-only setup inspection. `dry-run` remains Publication Mode language for running PullOps Operations without durable publication, while setup commands are reconcilers that either verify the desired setup state or apply it.

`pullops setup * --check` exits zero only when the setup area is satisfied. `pullops setup doctor` is also read-only and exits zero only when required runtime readiness blockers are absent; optional planning gaps such as missing `to-prd` or triage labels are warnings rather than blockers. Optional planning and authoring skill availability is detected from repo-local `.agents/skills/*/SKILL.md` files rather than host-provided model invocation metadata, because some usable skills deliberately disable direct model invocation. When optional Matt Pocock authoring skills are missing, PullOps reports the suggested installation command `npx skills@latest add mattpocock/skills` without running it.
