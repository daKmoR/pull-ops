# Support the Claude Code CLI as a runner

PullOps supports the Claude Code CLI alongside the Codex CLI without adding a new Runner Adapter: the configured Runner Command names the agent CLI, while `codex-cli` stays the inline adapter and `external` stays the handoff adapter. Runner Commands whose executable is `claude` run Claude Code headless: the runner ensures a `--print` flag, appends `--model` and the prompt as the final positional argument, relies on the spawn working directory instead of Codex's `-C` flag, and reads the final message from stdout instead of `--output-last-message`. When the Runner Command runs `claude` and `runner.models` is not overridden, the global Model Tiers default to `claude-opus-4-8` (high), `claude-sonnet-5` (mid), and `claude-haiku-4-5` (low) so tier-based operation defaults stay valid without per-repository model maps. Generated GitHub Actions workflows keep the same prepare/runner/complete shape and swap only the runner steps: the Claude Code Action (`anthropics/claude-code-action@v1`) replaces the Codex Action, `ANTHROPIC_API_KEY` replaces `OPENAI_API_KEY` in both the verify step and setup doctor's required-secret inspection, the action prompt instructs Claude to follow the prepared `runner_prompt.md` (the worker prompt already owns writing `runner_output.json`), and `--dangerously-skip-permissions` mirrors the Codex Action's `approval_policy=never` for unattended runs. Local runs started inside a Claude Code session (`CLAUDECODE` env) hand runner work back to the hosting agent through the `external` adapter, matching the existing Codex-hosted (`CODEX_THREAD_ID`) behavior.
