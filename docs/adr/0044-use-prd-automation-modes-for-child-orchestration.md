# Use PRD automation modes for child orchestration

PullOps uses `pullops:prd:auto-advance` and `pullops:prd:auto-complete` as durable PRD automation mode labels. Auto-advance prepares the PRD and drains the currently unblocked Child Issue frontier through the existing issue and PR operation labels while preserving human control over each Child Issue PR; later human Child Issue PR merges close issues and can unlock the next frontier for a later execution. Auto-complete is the hands-off PRD completion mode: it keeps selecting unblocked Child Issues, runs their implementation, automated review, and finalization gates, integrates finalized Child Issue work into the Umbrella Branch, and continues through dependency frontiers until the PRD branch is complete or blocked. Both modes preserve Child Issue PRs as traceable implementation units, and auto-complete keeps the final Umbrella PR merge to the default branch under human control. Local auto-complete keeps the CLI-wide dry-run default, but that dry-run should simulate the full PRD completion path locally, and virtually completed Child Issues should satisfy downstream Issue Dependencies in the same dry-run.
