# Use explicit workflow-facing commands

PullOps workflows call explicit Workflow-Facing Commands such as `pullops run prepare-prd --issue <number>`, `pullops run implement-issue --issue <number>`, `pullops run coordinate-prd --issue <number>`, and `pullops run review-pr --pr <number>`. Explicit commands keep workflow logs and YAML readable, while the full `pullops init` command can wait until the dogfood Workflow Kit has proven the operation behavior. The Target Repository should install `@pull-ops/cli` as a normal dependency or dev dependency, so workflows can run the local package after the repository's normal install step instead of globally installing PullOps.
