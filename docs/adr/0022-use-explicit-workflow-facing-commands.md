# Use explicit workflow-facing commands

PullOps workflows call explicit Workflow-Facing Commands such as `pullops run prd-prepare --issue <number>`, `pullops run prd-auto-advance --issue <number>`, `pullops run prd-auto-complete --issue <number>`, `pullops run issue-implement --issue <number>`, `pullops run pr-review --pr <number>`, and `pullops run pr-close-child-issue --pr <number>`. When GitHub Actions must run an adapter as a separate workflow step, workflows pass lifecycle and runner arguments explicitly, such as `pullops run issue-implement --phase prepare --runner codex-action --issue <number>` and `pullops run issue-implement --phase finalize --runner codex-action --runner-ran <true|false> --issue <number>`. Explicit commands keep workflow logs and YAML readable, while the full `pullops init` command can wait until the dogfood Workflow Kit has proven the operation behavior. The Target Repository should install `@pull-ops/cli` as a normal dependency or dev dependency, so generated workflows can run the package CLI after the repository's normal install step instead of globally installing PullOps; this repository may dogfood with `node src/cli/cli.js` before packaging.
