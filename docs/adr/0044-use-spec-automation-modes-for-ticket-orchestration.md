# Use Spec automation modes for ticket orchestration

PullOps uses `pullops:spec:auto-advance` and `pullops:spec:auto-complete` as durable Spec automation mode labels. Auto-advance prepares the Spec and drains the currently unblocked Ticket frontier through the existing issue and PR operation labels while preserving human control over each Ticket PR; later human Ticket PR merges close issues and can unlock the next frontier for a later execution. Auto-complete is the hands-off Spec completion mode: it keeps selecting unblocked Tickets, runs their implementation, automated review, and finalization gates, integrates finalized Ticket work into the Umbrella Branch, and continues through dependency frontiers until the Spec branch is complete or blocked. Both modes preserve Ticket PRs as traceable implementation units, and auto-complete keeps the final Umbrella PR merge to the default branch under human control. Local auto-complete keeps the CLI-wide dry-run default, but that dry-run should simulate the full Spec completion path locally, and virtually completed Tickets should satisfy downstream Issue Dependencies in the same dry-run.
