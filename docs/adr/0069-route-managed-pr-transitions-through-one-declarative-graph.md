# Route managed PR transitions through one declarative graph

One declarative PullOps-Managed PR Transition Graph in `src/managed-pr/transitionPolicy.js` answers every routing question of the pre-human review workflow: which operation follows which outcome, which outcome kinds each operation may report, which blocked outcomes re-add a follow-up Operation Label, and which operation continues an idle managed PR from recorded state. Before this decision those answers lived in five transition creators inside ManagedPrState, a separate state-based chooser, and private copies in PRD Child Coordination's umbrella loop and local operation selectors — changing one edge meant finding every copy.

ManagedPrState keeps PullOps-Managed PR Transition mechanics per ADR-0045 — PR State Marker parsing and updates, label mutations, failure comments — and consults the graph for routing. PRD Child Coordination consults the same graph through a local-drive routing profile instead of keeping its own sequence, so the child PR workflow is defined once. This keeps ADR-0049's split intact: the Operation Catalog owns identity facts and no routing, and routing policy stays with `src/managed-pr/`, the module that owns managed PR workflow state; what this decision removes are the copies that had leaked into sibling modules.

The graph is the routing trust boundary from ADR-0066: which operation may follow which outcome is verified harness structure that must not drift with runner quality, so it stays code-owned data rather than prompt policy. Run Budget and progress gates from ADR-0067 stay deterministic code whose results feed routing as recorded state, and graph predicates are limited to recorded workflow state — the only one today ends the workflow when an approving review validated an already-finalized tree. An outcome kind the graph has no edge for throws at transition creation, so a routing typo can never move a managed PR. Graph tests assert every edge lands on a workflow operation or terminal state and that success edges cannot loop without passing through pr-review's budget gates.
