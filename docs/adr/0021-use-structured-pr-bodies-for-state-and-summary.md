# Use structured PR bodies for state and summary

PullOps treats PR bodies as structured documents that it updates in place. The body carries human-facing summary sections, traceability such as `Closes #123`, `Refs #123`, and `Part of #456`, and the visible PullOps state including cycle counts, Trigger Context, runner task, model, and last operation; comments are reserved for failures, review threads, and notable handoffs.
