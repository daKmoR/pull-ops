# Use deterministic pr-prepare-merge for issue PRs

PullOps prepares standalone Concrete Issue PRs and Child Issue PRs for human rebase merge through deterministic checks and git rewriting, not through an AI-authored Commit Plan. This supersedes ADR-0035 for these issue PR paths: `pr-review` records the approved tree, `pr-prepare-merge` refuses stale or unmanaged state, validates native issue relationships against the configured branch shape, rewrites the reviewed tree to one traceable commit, records prepared tree/head markers and `Merge method: rebase`, waits for prepared-head checks, and only then removes draft status and PullOps labels.

Standalone Concrete Issue PRs target the default branch and keep default-branch closing references in the PR body. Child Issue PRs target their Parent Issue Umbrella Branch, use non-closing `Refs #<child>` and `Part of #<prd>` PR body references, and write deterministic `Refs: #<child>` plus `PRD: #<prd>` commit footers. Prepare Merge derives the child/parent relationship from GitHub native issue metadata, not PR body relationship text.
