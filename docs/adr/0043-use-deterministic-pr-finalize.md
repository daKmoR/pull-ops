# Use deterministic pr-finalize for issue PRs

PullOps finalizes standalone Concrete Issue PRs and Child Issue PRs for human rebase merge through deterministic checks and git rewriting. `pr-review` records the approved tree; `pr-finalize` refuses stale or unmanaged state, validates native issue relationships against the configured branch shape, rewrites the reviewed tree to one traceable commit, records finalized tree/head markers and `Merge method: rebase`, waits for pending finalized-head checks when GitHub reports them, treats absent finalized-head checks as no additional final check signal, and only then removes draft status and PullOps labels.

Standalone Concrete Issue PRs target the default branch and keep default-branch closing references in the PR body. Child Issue PRs target their Parent Issue Umbrella Branch, use non-closing `Refs #<child>` and `Part of #<prd>` PR body references, and write deterministic `Refs: #<child>` plus `PRD: #<prd>` commit footers. PR Finalize derives the child/parent relationship from GitHub native issue metadata, not PR body relationship text.
